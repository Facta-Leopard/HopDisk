HopDisk Support

HopDisk is designed to help you inspect storage mismatches and move only explicitly selected cleanup candidates to Trash. This page covers common support questions and how to report a problem.

Get help

For bug reports, questions, and support requests, use the public GitHub issue tracker: github.com/Facta-Leopard/HopDisk/issues.

When reporting an issue, include your macOS version, HopDisk version, scan type, and what you expected to happen. Do not post private file paths, file names, screenshots, or logs unless you are comfortable sharing them publicly.

Cleanup safety

  • HopDisk moves selected cleanup items to Trash only.
  • HopDisk does not empty Trash automatically.
  • HopDisk does not use root privileges, privileged helpers, or background daemons.
  • System-managed metadata, recent active files, virtual machines, and app data require review or are blocked.

System Data still looks large

macOS may count APFS snapshots, purgeable space, Spotlight indexes, package metadata, caches, and app support data under System Data. Some of those items are not safe cleanup targets. HopDisk separates safe candidates from protected or macOS-managed data so you can choose the right next step.

Folder access

HopDisk can scan only locations you select in the macOS file picker or saved locations you explicitly remember. If a scan reports inaccessible items, choose a narrower folder, reauthorize the saved location, or run a broader scan by selecting the relevant volume yourself.

Privacy

HopDisk runs locally on your Mac and does not upload scan results. Read the full Privacy Policy.
